# Modmux Branding Assets

## Brand Essence

- **Architectural**: a foundational layer that other tools rely on
- **Minimal**: clean, quiet, and unobtrusive
- **Modern**: built for the emerging multi-agent ecosystem
- **Trustworthy**: stable, predictable, and developer-friendly

## Typography

- **Headings**: Space Grotesk (400, 500, 700)
- **Body/Code**: IBM Plex Mono (400, 500, 600)

## Logo Concepts

Modmux's visual identity should reflect:

- **Layering**: stacked blocks or lines
- **Routing**: paths converging into one
- **Symmetry**: the mirrored "m" and "x" shapes
- **Modularity**: interlocking components

### Shape Language

- Rectilinear forms
- Soft corners (to avoid harshness)
- Horizontal flow (left → right routing)

## Voice & Messaging

Modmux should "speak" like:

- A system component, not a chatbot
- A trusted operator, not a hype machine
- A calm guide, not a verbose narrator

### Voice Guidelines

- Use short, declarative sentences
- Prefer verbs like route, unify, normalize, connect, serve
- Avoid emotional or marketing language
- Avoid metaphors unless they clarify architecture
